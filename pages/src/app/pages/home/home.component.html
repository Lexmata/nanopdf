use nanopdf::{Document, Pixmap};

let doc = Document::open("document.pdf")?;
let page = doc.load_page(0)?;
let pixmap = page.render(2.0)?; // 2x scale
pixmap.save_png("page.png")?;

Why NanoPDF?

Built for performance, designed for developers

Blazing Fast

Pure Rust implementation with optional parallel processing via Rayon. Up to 3x faster than MuPDF for batch operations.

Memory Safe

Zero unsafe Rust in core logic. Handle-based FFI for cross-language bindings eliminates memory leaks and use-after-free bugs.

MuPDF Compatible

Drop-in replacement for MuPDF. Same C headers, same API. Migrate existing projects without code changes.

Multi-Language

Native bindings for Rust, JavaScript (Node.js), and Go. Use the same powerful API in your favorite language.

Async Ready

Optional async I/O with Tokio for non-blocking operations. Perfect for web servers and high-throughput applications.

Full Compression

Support for all PDF compression filters: Flate, LZW, ASCII85, ASCIIHex, RunLength, JBIG2, JPEG2000, and more.
