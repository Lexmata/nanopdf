JavaScript Documentation

Node.js bindings for NanoPDF with full TypeScript support and async/await API.

Installation

# pnpm (recommended)
pnpm add nanopdf

# npm
npm install nanopdf

# yarn
yarn add nanopdf

The package includes prebuilt binaries for:

  • Linux (x64, arm64)
  • macOS (x64, arm64)
  • Windows (x64)

Basic Usage

import { Document } from 'nanopdf';

// Open a PDF document
const doc = await Document.open('document.pdf');

// Document info
console.log('Pages:', doc.pageCount);
console.log('Title:', doc.title);
console.log('Author:', doc.author);

// Load and render a page
const page = await doc.loadPage(0);
const pixmap = await page.render(2.0);

// Save as PNG
await pixmap.savePng('page-0.png');

// Get PNG as Buffer
const pngBuffer = await pixmap.toPng();

// Clean up
doc.close();

TypeScript Types

NanoPDF ships with complete TypeScript definitions:

import {
  Document,
  Page,
  Pixmap,
  Point,
  Rect,
  Matrix,
  NanoPDFError,
  ErrorCode
} from 'nanopdf';

// All types are fully typed
const bounds: Rect = page.bounds;
const width: number = pixmap.width;
const samples: Uint8Array = pixmap.samples;

Error Handling

import { Document, NanoPDFError, ErrorCode } from 'nanopdf';

try {
  const doc = await Document.open('file.pdf');
} catch (error) {
  if (error instanceof NanoPDFError) {
    switch (error.code) {
      case ErrorCode.FileNotFound:
        console.error('File not found');
        break;
      case ErrorCode.PasswordRequired:
        console.error('Password required');
        break;
      case ErrorCode.InvalidPdf:
        console.error('Invalid PDF');
        break;
      default:
        console.error('Error:', error.message);
    }
  }
}
