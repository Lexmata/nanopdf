Getting Started with NanoPDF

NanoPDF is a high-performance PDF library available for Rust, JavaScript, and Go. This guide will help you install and start using NanoPDF in your projects.

Installation

Rust

Add NanoPDF to your Cargo.toml:

[dependencies]
nanopdf = "0.1"

# Optional features
nanopdf = { version = "0.1", features = ["parallel", "async"] }

JavaScript (Node.js)

Install via pnpm (recommended), npm, or yarn:

# pnpm (recommended)
pnpm add nanopdf

# npm
npm install nanopdf

# yarn
yarn add nanopdf

Go

Install using Go modules:

go get github.com/lexmata/nanopdf/go-nanopdf

Quick Start

Opening a PDF Document

Rust 
use nanopdf::Document;

fn main() -> nanopdf::Result<()> {
    let doc = Document::open("document.pdf")?;
    
    println!("Pages: {}", doc.page_count());
    println!("Title: {:?}", doc.title());
    
    Ok(())
}
JavaScript 
import { Document } from 'nanopdf';

const doc = await Document.open('document.pdf');

console.log('Pages:', doc.pageCount);
console.log('Title:', doc.title);

doc.close();
Go 
package main

import (
    "fmt"
    nanopdf "github.com/lexmata/nanopdf/go-nanopdf"
)

func main() {
    doc, err := nanopdf.OpenDocument("document.pdf")
    if err != nil {
        panic(err)
    }
    defer doc.Close()

    fmt.Printf("Pages: %d\n", doc.PageCount())
    fmt.Printf("Title: %s\n", doc.Title())
}

Rendering Pages

Render PDF pages to images with customizable resolution:

Rust 
use nanopdf::{Document, Pixmap};

let doc = Document::open("document.pdf")?;
let page = doc.load_page(0)?;

// Render at 2x scale (144 DPI)
let pixmap = page.render(2.0)?;

// Save as PNG
pixmap.save_png("page-0.png")?;

Optional Features

NanoPDF supports several optional features that can be enabled in Rust:

parallel Enable parallel processing with Rayon for batch operations
async Enable async I/O with Tokio for non-blocking operations
jpeg2000 Enable JPEG2000 image support
jbig2 Enable JBIG2 image support
